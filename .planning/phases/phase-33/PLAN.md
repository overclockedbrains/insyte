# Phase 33 — Community Gallery

**Goal:** A public `/community/gallery` page showing all AI-generated scenes created by any user. Browsable grid sorted by recency or popularity. No user data exposed — not even indirectly.

**Status:** Planned April 19, 2026.

**Prerequisite:** Phase 32 ✅ (stable pipeline) · Phase 11 ✅ (`user_generated_scenes` table exists)

---

## Route structure

`/community` is the base — kept as a layout/index shell so future sub-routes can live alongside:

```
/community                  → redirect or index (future: leaderboard, challenges, etc.)
/community/gallery          → this phase — AI-generated scenes feed
/community/...              → reserved for future sub-routes
```

---

## Privacy design

We query `user_generated_scenes` joined to `scenes` — but we **only select non-PII columns**:
`scene_slug, query, generated_at` from `user_generated_scenes` + `title, type, hit_count` from `scenes`.

`user_id` is never selected, never returned, never logged. The join is read-only and server-side (service role), so the anon RLS policy on `user_generated_scenes` is not a blocker.

Pre-built scenes are naturally excluded because they are never inserted into `user_generated_scenes` — no slug filter needed.

---

## Data source

```sql
SELECT ugs.scene_slug, ugs.query, ugs.generated_at,
       s.title, s.type, s.hit_count
FROM user_generated_scenes ugs
JOIN scenes s ON s.slug = ugs.scene_slug
ORDER BY ugs.generated_at DESC   -- or s.hit_count DESC for popular
LIMIT 20 OFFSET ?
```

Uses `getServerSupabase()` (service role) — bypasses the `user_generated_scenes` authenticated-only RLS.

## `CommunityScene` response type

```ts
interface CommunityScene {
  slug: string
  title: string
  type: string        // concept | dsa | lld | hld
  query: string       // user's original prompt — shown as card subtitle
  hit_count: number
  generated_at: string  // ISO timestamp — formatted as relative time client-side
}
```

---

## API route design (`GAL-01`)

- `getServerSupabase()` — service role bypasses `user_generated_scenes` RLS
- Supabase query: `.from('user_generated_scenes').select('scene_slug, query, generated_at, scenes(title, type, hit_count)').order(...)` using Supabase's nested select
- `sort=recent` → `.order('generated_at', { ascending: false })`
- `sort=popular` → join orders by `scenes.hit_count` — use `.order('scenes(hit_count)', { ascending: false })` or a raw RPC if Supabase nested ordering is unsupported
- `page` → `.range(page * 20, page * 20 + 19)`
- `hasMore` = `data.length === 20`

## Page UI spec

- Header: "Community Gallery" + subtitle "Simulations generated by the community"
- Sort toggle: `Recent | Popular` pill tabs (Framer Motion underline indicator)
- Grid: `grid-cols-2 md:grid-cols-3 lg:grid-cols-4`
- Each card: scene title, query prompt as subtitle (truncated to 60 chars), type badge, relative time, hit count pill
- "Load more" button — appends next page, no IntersectionObserver
- Relative time via `Intl.RelativeTimeFormat` — no extra dependency
- Glass-panel + glow-border on hover, consistent with Explore cards
