# ISCL Quick Reference

ISCL (Insyte Scene Language) is the text DSL generated by Stage 1 of the AI pipeline. This is a quick-reference for understanding pipeline prompts, debugging AI output, and writing test fixtures.

For the full grammar spec see `packages/scene-engine/src/iscl/parser.ts`.

---

## Full Script Template

```
SCENE "How does Binary Search work?"
TYPE dsa-trace
LAYOUT code-left-canvas-right

# --- Visuals ---
VISUAL arr array HINT linear-H
VISUAL ptr text-badge SLOT top-center

# --- Steps ---
STEP 0 : init
STEP 1 : SET arr items=[1,3,5,7,9,11] | SET ptr label="lo=0 hi=5"
STEP 2 : SET arr highlight=[2] | SET ptr label="mid=2 → 5"
STEP 3 : SET arr highlight=[3,4,5] | SET ptr label="lo=3 hi=5"

# --- Explanation ---
EXPLANATION
1: The array is initialized with 6 sorted elements.
2: Mid-point is index 2. Value 5 is less than target 7.
3: Left half discarded. New search window is indices 3-5.

# --- Popups ---
POPUP arr 2 "Mid computed at index 2"

# --- Challenges ---
CHALLENGES
1: What is the time complexity of binary search? | O(log n) | O(n) | O(1) | O(n²) | 0

# --- Controls ---
CONTROL toggle-group speed "Playback Speed" arr
```

## Block Reference

### `SCENE`
```
SCENE "title string"
```
Sets `scene.title`. Must be a quoted string on the same line.

### `TYPE`
```
TYPE concept | dsa-trace | lld | hld
```

### `LAYOUT`
```
LAYOUT canvas-only | text-left-canvas-right | code-left-canvas-right
```

### `VISUAL`
```
VISUAL <id> <type> [HINT <layoutHint>] [SLOT <slotPosition>]
```
- `id` — unique identifier, used in all `SET` references
- `type` — must be a valid `VisualType`
- `HINT` — optional layout algorithm hint (see table below)
- `SLOT` — optional named position within the canvas (see table below)

### `STEP`
```
STEP <index> : <body>
```
- `index` must be monotonically increasing (0, 1, 2, …)
- `STEP 0 : init` is the required initialization step (no SET clauses)
- Body is one or more `SET` clauses separated by ` | `

### `SET`
```
SET <visualId> <key>=<value> [<key>=<value> ...]
```
- `visualId` must be declared as a VISUAL
- Multiple key-value pairs on the same `SET` clause are space-separated
- Array values use `[...]` JSON syntax: `items=[1,2,3]`

### `EXPLANATION`
```
EXPLANATION
<stepIndex>: <body text>
<stepIndex>: <body text>
```
Each entry appears when playback reaches `stepIndex`. One entry per line after the `EXPLANATION` keyword.

### `POPUP`
```
POPUP <visualId> <atStep> "message text"
```
Inline callout attached to a visual at a specific step.

### `CHALLENGES`
```
CHALLENGES
<index>: <question> | <opt0> | <opt1> | <opt2> | <opt3> | <correctIndex>
```
`correctIndex` is 0-based (0 = first option is correct).

### `CONTROLS`
```
CONTROL <type> <id> "<label>" <boundVisualId>
```

## Valid LayoutHint Values

| Value | Algorithm | Best for |
| --- | --- | --- |
| `dagre-TB` | Dagre top-to-bottom | Dependency graphs, call graphs |
| `dagre-LR` | Dagre left-to-right | Flowcharts, system diagrams |
| `dagre-BT` | Dagre bottom-to-top | Inverted hierarchies |
| `tree-RT` | d3-hierarchy radial | Recursion trees |
| `linear-H` | Horizontal arithmetic | Arrays, queues |
| `linear-V` | Vertical arithmetic | Stacks |
| `grid-2d` | Grid arithmetic | DP tables |
| `hashmap-buckets` | Bucket arithmetic | HashMaps |
| `radial` | Radial arithmetic | Concept webs |

## Valid SlotPosition Values

| Value | Location on canvas |
| --- | --- |
| `top-left` | Top-left quadrant |
| `top-center` | Top center |
| `top-right` | Top-right quadrant |
| `center-left` | Left-center |
| `center` | Center of canvas |
| `center-right` | Right-center |
| `bottom-left` | Bottom-left quadrant |
| `bottom-center` | Bottom center |
| `bottom-right` | Bottom-right quadrant |
| `canvas-left` | Left half of canvas |
| `canvas-right` | Right half of canvas |

## Known AI Hallucinations & Fixes

| Issue | Fix applied |
| --- | --- |
| Model wraps ISCL in ` ```iscl ``` ` fences | `stripCodeFences()` in `iscl-preprocess.ts` |
| Model splits a `STEP N :` body across multiple lines with bare `SET …` on next line | `joinStepContinuations()` in `iscl-preprocess.ts` |
| Model puts `SET` lines after `STEP 0 : init` | `joinStepContinuations` silently drops them (init steps cannot have SET clauses) |
| Model emits invalid `HINT` value (e.g. `linear-horizontal`) | Parser rejects → Stage 1 retried |
| Model emits invalid `SLOT` value | Parser rejects → Stage 1 retried |
| Model uses a visual ID in `SET` that wasn't declared in a `VISUAL` block | Parser rejects with cross-reference error → Stage 1 retried |

## Debugging Tips

1. **Set `PIPELINE_MAX_RETRIES=0`** in `.env.local` to fail fast and see the raw ISCL in server logs without burning tokens on retries.
2. The ISCL string that reaches the parser is logged at `[pipeline] Stage 1` with debug level.
3. `parseISCL(script)` returns `{ ok: false, error: ISCLParseError }` with a human-readable message pointing to the offending line — paste raw ISCL into a quick test file to debug without running the full pipeline.
4. Parser unit tests in `packages/scene-engine/src/iscl/parser.test.ts` cover all validation rules — add a failing case there before fixing the parser.
